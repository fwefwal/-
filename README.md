Вот готовое, максимально «ленивое», но при этом адекватное и профессиональное ТЗ, которое ты можешь сразу кинуть фронтендеру (или себе через неделю, когда проснёшься и захочешь начать). Всё разбито по этапам так, чтобы первые 3–5 дней нагрузка была почти нулевая.

### Техническое задание  
Проект: Лендинг + личный кабинет для небольшой бухгалтерской конторы (аутсорс-бухгалтерия и биржа бухгалтеров)

### Цель
Привлекать клиентов и давать существующим клиентам удобный кабинет (загрузка/скачивание документов, просмотр отчётов, чат с бухгалтером).

### Стек (рекомендуемый, но можно договориться)
- React 18 + Vite / Next.js (App Router) — на твой выбор
- TypeScript
- Tailwind CSS + shadcn/ui (чтобы не рисовать всё с нуля)
- Формы — React Hook Form + Zod
- Авторизация — пока простая email + пароль через Supabase Auth (или Clerk, если хочешь быстрее)

### Этапы (чтобы первые дни вообще ничего не делать)

Этап 0 — «День лени» (1–2 дня, нагрузка = 0)
- Сделать репозиторий
- Инициализировать проект (Vite + React + TS + Tailwind + shadcn)
- Залить пустой шаблон с главной страницей и надписью «В разработке»
- Деплой на Vercel/Netlify (чтобы уже был живой URL и заказчик успокоился)

Этап 1 — Главная страница (лендинг) — 3–4 дня, но реально делается за день
Блоки (сверху вниз):
1. Хедер с логотипом и кнопками «Войти» / «Оставить заявку»
2. Hero: заголовок «Бухгалтерия без головной боли», подзаголовок, кнопка «Оставить заявку»
3. Преимущества (4–6 карточек)
4. Как мы работаем (3–4 шага с иконками)
5. Тарифы (3 карточки: «Мини», «Стандарт», «Премиум»)
6. Отзывы (можно пока заглушки)
7. Форма заявки внизу + контакты
8. Футер

Этап 2 — Личный кабинет (MVP) — начинаем через неделю, нагрузка минимальная
Страницы и функционал (по приоритету):
1. /login и /register (email + пароль)
2. Dashboard — приветствие + последние загруженные документы (таблица)
3. Страница «Документы»
   - Загрузка файлов (drag & drop)
   - Список своих документов с фильтром по месяцам
   - Скачивание и удаление
4. Страница «Отчёты» — пока просто список PDF с возможностью скачать
5. Страница «Чат с бухгалтером» — простое поле ввода и сообщения (можно пока мок-данные)
6. Профиль и выход

Этап 3 — Админка для бухгалтера (делаем позже, когда будет хоть один клиент)
- Список всех клиентов
- Просмотр загруженных документов
- Загрузка готовых отчётов
- Чат

### Приоритеты и сроки (чтобы не напрягаться)
- Неделя 1: Этапы 0 и 1 (главная готова и уже на хостинге) → заказчик счастлив
- Неделя 2–3: Личный кабинет MVP (Этапы 2 пункты 1–4)
- Неделя 4+: чат, админка, красивости

### Дополнительно (можно кинуть потом)
- Адаптив под мобилку (Tailwind делает 90% сам)
- SEO-мета (title, description) на главной
- Фавиконка и логотип (пока заглушка)
- Google Analytics и Яндекс.Метрика

### Что отдаём заказчику после 1-й недели
Живую ссылку на красивый лендинг + форму заявки, которая пишет в Google Sheets или на почту.
